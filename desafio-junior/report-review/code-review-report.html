🔴 Requisito de Negócio Ausente: Campo 'logradouro' não é convertido para lowercase
Localização & Contexto

Achados Importantes

Importante Utils / Validador
⚠️ Sanitização ineficaz no método Validadores.validaCep
Localização & Contexto
Arquivo: utils/Validadores.java (linhas 11-16)
Descrição do Problema
O método calcula String cepNoNull = cep.replaceAll("\\D", "") para checar se possui 8 dígitos, mas ao final retorna a variável original cep sem sanitização. Se o usuário passar "01001-000", o Feign client receberá a string com hífen.
❌ Código Atual
String cepNoNull = cep.replaceAll("\\D", ""); if (cepNoNull.length() != 8) { throw new IllegalArgumentException("O CEP deve conter 8 caracteres numericos"); } return cep; // Retorna com hífen/caracteres originais
✅ Sugestão Corrigida
String cepApenasNumeros = cep.replaceAll("\\D", ""); if (cepApenasNumeros.length() != 8) { throw new IllegalArgumentException("O CEP deve conter 8 caracteres numéricos"); } return cepApenasNumeros; // Retorna sanitizado
Importante Nomenclatura / Convenção
⚠️ Nome de pacote fora das convenções Java: 'com.desafio.casebackend.DTOs'
Localização
Pacote: com.desafio.casebackend.DTOs
Descrição do Problema
Em Java, nomes de pacotes devem ser estritamente em minúsculas (ex: com.desafio.casebackend.dto). Utilizar maiúsculas (DTOs) viola a convenção padrão da linguagem.
Importante Error Handling
⚠️ Ausência de Handler para FeignException / Integração Externa
Localização
Arquivo: error/GlobalExceptionHandler.java
Descrição do Problema
Caso o ViaCEP esteja fora do ar ou o CEP não seja encontrado (HTTP 404/500 na origem), o Feign lança feign.FeignException, resultando em resposta HTTP 500 sem tratamento gracioso.
