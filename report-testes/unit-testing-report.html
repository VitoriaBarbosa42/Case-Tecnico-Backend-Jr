🎯 Revisão Proativa de Cenários (QAs Lead)

Com base na análise do refinamento e de segurança da aplicação, mapeamos os cenários a incluir e excluir:

Cenários Incluídos (Regras de Negócio e Cobertura)

  • ViaCEP Service - CEP não encontrado: Retorno da API com erro = true mapeado e tratado na camada de negócio, lançando erro correto ao cliente.
  • Controller - Mapeamento e Validação: Validação sintática do CEP (@Pattern e @NotBlank) testados diretamente na camada web via MockMvc.
  • DTO - CEP Geral (Logradouro Vazio): Casos de municípios pequenos com logradouro vazio/nulo não causam quebras nem exceções indesejadas no DTO.
  • BFF - Comunicação com API Externa: Tratamento global de erros para capturar falhas de timeout e conexões HTTP externas (FeignException).

Cenários Excluídos / Evitados (Code Smell & Fragilidade)

  • Validação precoce no Record: Excluída a validação de existência de CEP no construtor de ViaCepResponseDTO (violava SRP e impedia logradouros legítimos vazios).
  • Testes de injeção manual de dependência: Excluídos testes redundantes como shouldAcceptNullServiceInConstructor (não testam valor e geram poluição de código).
  • Tratamento de NPE genérico: Excluída a captura e máscara de NullPointerException no handler global (má prática que esconde bugs estruturais).
