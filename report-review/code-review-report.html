Importante BFF / Client
Feign Client sem configuração de Resiliência (Timeout / Circuit Breaker)
Problema: O client ViaCepClient consome um serviço público terceiro (ViaCEP) sem configuração explícita de timeout ou fallbacks. 💡 Sugestões de Melhoria (2)

Sugestão Exceptions
Substituir NullPointerException e criar exceções específicas de negócio
Melhoria: O GlobalExceptionHandler mapeia NullPointerException para retornar o texto "Cep invalido". O uso de NullPointerException para tratar regras de negócio ou validações é desencorajado por ser uma exceção de runtime muito genérica. Recomenda-se criar exceções personalizadas (ex: CepInvalidoException ou CepNaoEncontradoException) estendendo de RuntimeException.
Impacto
Manutenibilidade / Clean Code
Sugestão Controller
Remover duplicação de validação por regex
Melhoria: A validação de Regex por pattern de CEP no ViaCepController (@Pattern(regexp = "\\d{5}-?\\d{3}")) valida o CEP no nível HTTP, mas a regra de negócio e tamanho também são testados no nível de serviço e utilitário. O ideal é centralizar a expressão regular de validação do formato do CEP em um só lugar (por exemplo, na classe Validadores) para evitar inconsistência de formato caso as regras mudem futuramente.
Impacto
Manutenibilidade (DRY - Don't Repeat Yourself)
