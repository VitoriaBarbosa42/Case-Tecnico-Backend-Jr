Mudanças Principais - Itens Críticos

Crítico Error Handler
🔴 Construtor de ErrorResponse com assinatura inconsistente
Onde
Arquivo: error/ErrorResponse.java (linha 17)
Contexto: GlobalExceptionHandler linha 16-21
Problema
O construtor declarado com 6 parâmetros não está alinhado com como é chamado no GlobalExceptionHandler. O construtor recebe parâmetros duplicados e genéricos que não correspondem ao uso prático.
❌ Código Atual
// ErrorResponse.java public ErrorResponse(int value, String requisiçãoInválida, String message, String requestURI, String error, String path) { this.timestamp = LocalDateTime.now(); this.error = error; this.message = message; this.path = path; } // GlobalExceptionHandler.java - Chamada inconsistente new ErrorResponse( status.value(), "Requisição Inválida", ex.getMessage(), request.getRequestURI() // Faltam 2 parâmetros! );
✅ Sugestão
// Usar @AllArgsConstructor ou construtor simples @Data @AllArgsConstructor @NoArgsConstructor public class ErrorResponse { private LocalDateTime timestamp; private int status; private String error; private String message; private String path; } // Ou fazer construtor customizado: public ErrorResponse(int status, String error, String message, String path) { this.timestamp = LocalDateTime.now(); this.status = status; this.error = error; this.message = message; this.path = path; }
Violação SOLID
Single Responsibility: A classe tem responsabilidade dupla (model + lógica de inicialização)
Open/Closed: Difícil de estender sem modificar o construtor

Crítico Validador
🔴 Uso de NullPointerException para validação é impropriedade
Onde
Arquivo: utils/Validadores.java (linha 7-9)
Contexto: Método validaCep()
Problema
NullPointerException é uma exception do sistema, não deve ser lançada por código de negócio. A prática correta é lançar IllegalArgumentException ou criar uma exceção customizada para validação.
❌ Código Atual
if (cep == null || cep.isBlank()) { throw new NullPointerException( "O CEP não pode ser nulo nem estar em branco" ); }
✅ Sugestão
if (cep == null || cep.isBlank()) { throw new IllegalArgumentException( "O CEP não pode ser nulo nem estar em branco" ); }
Por que importa
• Clean Code: Exceptions específicas facilitam tratamento correto
• Testabilidade: Testes podem validar exceção esperada
• GlobalExceptionHandler: Já trata IllegalArgumentException, não precisa adicionar handler

Crítico Controller
🔴 Endpoint redundante `GET /viacep/` viola Single Responsibility
Onde
Arquivo: controller/ViaCepController.java (linha 22-26)
Método: cepNull()
Problema
O método cepNull() é semanticamente estranho: passa null para o serviço que imediatamente vai lançar exceção. Viola princípio de design: não deve haver endpoint que apenas retorna erro de validação.
❌ Código Atual
@GetMapping("/") public ResponseEntity cepNull(){ return ResponseEntity.ok( viaCepService.buscaPorCep(null) ); } // Sempre retorna erro: // GET /viacep/ → 400 Bad Request
✅ Sugestão
// Remover este método completamente! // Se deseja documentação, adicione: @GetMapping("/") public ResponseEntity> info() { return ResponseEntity.ok(Map.of( "message", "Use /viacep/{cep}", "example", "/viacep/01001000" )); }
Violação SOLID
Single Responsibility: Endpoint deve ter um propósito claro
Open/Closed: Endpoint quebrado dificulta futuras extensões

Importante Validador
⚠️ Validador retorna CEP sem formatação, contrato retorna com formatação
Onde
Validadores.java linha 16 retorna CEP sem formatação
DTO mostra CEP como "01001-000" (formatado)
Problema
Inconsistência entre o que é validado e o que é retornado. O DTO da API externa retorna formatado ("01001-000"), mas o validador não garante que o CEP armazenado mantém esta formatação.
Fluxo Atual
// Entrada: "01001000" Validadores.validaCep("01001000") ↓ remove \D (digits only) ↓ valida length ↓ retorna "01001000" // SEM formatação! // Mas DTO retorna: "01001-000" // COM formatação
✅ Solução
// Formatter deve estar na resposta // ou antes de armazenar public static String formatarCep(String cep) { return cep.replaceAll( "(\\d{5})(\\d{3})", "$1-$2" ); } // Usar em DTO ou Service
